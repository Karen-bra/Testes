## Requisitos do Projeto



### Requisitos Funcionais



|ID    | Descrição                | Prioridade |
|-------|---------------------------------|----|
| RF-01 | O sistema deve permitir cadastro de usuários (Professores) | Alta| 
| RF-02 | O sistema deve permitir o login de usuários com e-mail e senha. | Alta|
| RF-03 | O sistema deve permitir sair da conta.                        | Média|
| RF-04 | O sistema deve permitir o cadastro de alunos.                  |Baixa|
| RF-05 | O sistema deve permitir o lançamaento de duas notas por aluno.  | Média|
| RF-06 | O sistema deve calcular automaticamente a média do aluno.      | Alta|
| RF-07 | O sistema deve classificar o aluno como Aprovado, Recuperação e Reprovado | Alta|
| RF-08 | O sistema deve permitir a listagem de todos os alunos cadastrados.|Alta|
| RF-09 | O sistema deve permitir a edição dos dados do aluno (Nome e Notas). |Alta|
| RF-10 | O sistema deve exibir a situação do aluno na tabela | Média|
| RF-11 | O sistema deve permitir a exclusão de alunos.| Baixa|
| RF-12 | O sistema deve armazenar os dados localmente (localstorage). | Baixa|
| RF-13 | O sistemea deve permitir a busca de alunos pelo nome. | Alta|


**Prioridade: Alta / Média / Baixa. 

### Requisitos não Funcionais


|ID      | Descrição               |Prioridade |
|--------|-------------------------|----|
| RNF-01 | O sistema deve possuir uma interface simples e intuitiva.| Alta| 
| RNF-02 | O sistema deve funcionar nos principais navegadores (Chrome, Edge,Firefox).| Alta| 
| RNF-03 | O sistema deve ter resposta rápida, sem travamentos. | Média|
| RNF-04 | O sistema deve garantir a segurança básica dos dados dos usuário. | Alta|
| RNF-05 | O sistema deve ser desenvolvido em HTML, CSS e JavaScript. | Média|
| RNF-06 | O sistema deve ser responsivo (adaptável a diferentes tamanhos de tela). | Alta|
| RNF-07 | O sistema deve ter boa usabilidade e acessibilidade | Baixa|

**Prioridade: Alta / Média / Baixa.
